Logik und Mathematische Aussagen

Operatoren

¬ : Negation (NICHT)
^ : Konjunktion (UND)
v : Disjunktion (ODER)
=> : Implikation (WENN-DANN)
<=> : Äquivalenz (GENAU DANN WENN)

Wahrheitstafel

Eine Wahrheitstafel ist wie folgt aufgebaut:

DNF (Disjunktive Normalform)

Definition:
Ein Term D ist in disjunktiver Normalform (DNF) bezüglich der Variablen A1, ... , An, wenn

D = C1 v C2 v ... v Ck

wobei Cj = B1 ^ B2 ^ ... ^ Bm für j = 1, ... , k paarweise verschieden und
Bi = Ai oder Bi = ¬Ai für i = 1, ... , n

Was bedeutet das?

"Das sind Oder-Verknüpfungen und UND-Ketten
und dienen einer simpleren Darstellung für Logik Aussagen"

Beispiel:

Gegeben: (A v B) ^ C <=> A v C

Aufgabe: Bestimme unter Angabe der zugehörigen Wahrheitstafel eine DNF für die Aussage.

Lösungsansatz: Stelle Wahrheitstafel auf. Für alle Wertepaare, bei denen
(A v B) ^ C <=> A v C wahr ist, schreibe die Wertepaare in DNF-Form auf.

Beweismethoden

Vollständige Induktion

Eine vollständige Induktion ist in folgende Schritte eingeteilt:

1. Induktionsbasis: Beweise, dass die Aussage für n = 1 gilt.
2. Induktionsschritt: (k -> k + 1)

    2.1 Annahme: "Für ein beliebiges, aber festes k ∈ lN sei A(k) wahr.
    2.2 Zu zeigen: Beweise, dass A(k) -> A(k + 1) gilt.
    2.3 Induktionsbeweis: Rechnung

Beispiel:

Behauptung: Für alle n ∈ N gilt: ∑nj=1 j = 1 + 2 + ... + n = n (n + 1)/ 2
Aufgabe: Beweise dies mittels vollständiger Induktion.

Relationen

Äquivalenzrelationen

Definition:
Eine Relation R auf einer Menge M ist eine Äquivalenzrelation,
wenn sie reflexiv, symmetrisch und transitiv ist.

Beispiel:
Zeige: R = {(x, y) ∈ N x N | x + y ist gerade} ist eine Äquivalenzrelation.

Z.Z.: R reflexiv, symmetrisch und transitiv.

1. Reflexiv

Z.Z.: ∀x ∈ N:
(x,x) ∈ R
(aka. xRx)

Sei x ∈ N.
x + x = 2x, also gerade und (x, x) ∈ R

2. Symmetrisch

Z.Z.: ∀x,y ∈ N:
(x,y) ∈ R, dann auch (y,x) ∈ R
(aka. xRy => yRx)

Seien x, y ∈ N mit (x, y) ∈ R.
Dann ∃k ∈ N: x + y = 2k
Somit ist auch y + x = 2k und damit (y, x) ∈ R

3. Transitiv

Z.Z.: ∀x,y,z ∈ N:
(x,y) ∈ R und (y,z) ∈ R, dann auch (x,z) ∈ R
(aka. xRy & yRz => xRz)

Seien x, y, z ∈ N mit (x, y) ∈ R und (y, z) ∈ R.
Dann ∃k, l ∈ N: x + y = 2k und y + z = 2l
Somit ist:

(x + y) + (y + z) = 2k + 2l
<=> x + 2y + z = 2k + 2l
<=> x + 2y + z = 2(k + l)
<=> x + z = 2(k + l - y)
<=> x + z = 2m
d.h. (x, z) ∈ R

Ordnungsrelationen

Definition: Eine Relation R auf einer Menge M ist eine Ordnungsrelation,
wenn sie reflexiv, anti-symmetrisch und transitiv ist.

Anti-Symmetrie

Es gilt: xRy & yRx (Siehe Symmetrie)

xRy - yRx = 0
(x + 3y) - (y + 3x) = 4k1 - 4k2
<=> -2x + 2y = 4k1 - 4k2
<=> -x + y = 2(k1 - k2)

Falls k1 = 1, k2 = 0
Dann: -x + y = 2(1 - 0)
<=> -x + y = 2
<=> -x = 2 - y
<=> = y - 2 =/ x

Abbildungen

Abbildungen von Menge A und Menge B werden wie folgt dargestellt:

Abbildungen haben dabei folgende Eigenschaften:

Injektiv

Eine Abbildung ist injektiv, wenn jedes Element aus A auf ein Element aus B abgebildet wird.
Zu Zeigen ist dass für alle a1 und a2 aus A gilt: f(a1) ≠ f(a2) => a1 ≠ a2

Beispiel:
Ist f injektiv?
∀x1,x2 ∈ M = lR\{1}:
f(x1) = f(x2) => x1 = x2

Seien x1, x2 ∈ lR\{1} mit f(x1) = f(x2)
f = 1 / x - 1

Z.Z.: x1 = x2
Dann gilt
1 / x1 - 1 = 1 / x2 - 1
x1 - 1 = x2 - 1, d.h.
x1 = x2
Somit ist f injektiv.

Surjektiv

Eine Abbildung ist surjektiv, wenn jedes Element aus B mindestens einmal abgebildet wird.
Zu Zeigen ist dass für jedes b aus B ein a aus A existiert, sodass f(a) = b

Beispiel:
Ist f surjektiv?
∀y ∈ N = lR: ∃x ∈ M = lR\{1}: f(x) = y

Sei y ∈ lR. Finde x ∈ lR\{1}, sodass f(x) = y,
d.h. 1 / x - 1 = y

Löse nach x auf:
1 / x - 1 = y
(x - 1) * y = 1
xy - y = 1
xy = y + 1
x = y + 1 / y

Nur wenn y /= 0, dann ist x = y + 1 / y ∈ lR\{1} definiert.
Also ist f nicht surjektiv.

Also f nicht bijektiv.

RSA-Kryptosystem

Wir verfahren beim RSA-Kryptosystem wie folgt:
(dies variiert je nach Aufgabenstellung)

1. Schlüsselerzeugung

• Wähle zwei Primzahlen p und q
• Berechne N = p * q
• Berechne φ(N) = (p - 1) * (q - 1)
• Wähle ein zufälliges e ∈ lN mit:
        1 < e < φ(N) und
        ggT(e, φ(N)) = 1
• Berechne d ∈ lN (das Inverse von e) mit:
        e * d ≡ 1 (mod φ(N))
        -> Nutze eeA von (e, φ(N))

 -> Öffentlicher Schlüssel: (e, N)
 -> Privat: d, p, q, φ(N)
2. Verschlüsselung

• Wähle Nachricht M ∈ lN mit 0 <= m < N
• Berechne C = Me (mod N)

3. Entschlüsselung

• Berechne M = Cd (mod N)
        -> Teile in Primfaktorzerlegung auf. (N -> p & q)
        -> Löse mittels CRS

Beispiel:

Aufgabe: Es soll die Nachricht C = 48 entschlüsselt werden.
a) Bestimme Inverse d zu e
b) Bestimme original Nachricht M

Gegeben: (e, N) = (77, 143)

Lösung:

a)
Primfaktorzerlegung:
Zerlege N in Primzahlen:

143 = 11 * 13
φ(143) = (11 - 1) * (13 - 1) = 120

eeA (e, φ(N))
Nach Bézout erhalten wir:
ggT(77, 120) = 1 = -34 * 120 + 53 * 77
Also d = 53

b)
Entschlüsselung:
M = Cd (mod N)
M = 4853 (mod 143)

Nach Primfaktorzerlegung erhalten wir folgendes Kongruenzsystem:

1. M ≡ 4853 (mod 11)
2. M ≡ 4853 (mod 13)
1. M ≡ 4853 ≡ 453 = (410)5 * 43 ≡* 43 = 64 ≡ 9 (mod 11)
2. M ≡ 4853 ≡ (-4)53 = (-412)4 * (-4)5 ≡* (-4)5 = -1024 ≡ -10 ≡ 3 (mod 13)
*: Euler

Chinesischer Restsatz:
Da ggT(11, 13) = 1, ist CRS anwendbar:

1. M ≡ 9 (mod 11)
2. M ≡ 3 (mod 13)

a1 = 1, m1 = 11, N1 = 13
a2 = 1, m2 = 13, N2 = 11

1. N1 * y1 ≡ 1 (mod m1)
2. N2 * y2 ≡ 1 (mod m2)

1.
13y1 ≡ 1 (mod 11)
2y1 ≡ 1 (mod 11)
2y1 + 11Y = 1
-> eeA: (11, 2) -> y1 = -5

2.
11y2 ≡ 1 (mod 13)
11y2 + 13Y = 1
-> eeA: (13, 11) -> y2 = 6

Also y1 = -5 und y2 = 6

Nach Bézout gilt: ggt(13, 11) = 1 = -5 * 13 + 6 * 11
Partikuläre Lösung

M0 = a1y1N1 + a2y2N2

M0 = 9 * (-5) * 13 + 3 * 6 * 11
       = -585 + 198
       = -387 ≡ 42 (mod 143)

Also ist M = 42.
